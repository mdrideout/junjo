Whether you are building a chatbot, a data-processing pipeline, or a stateful
workflow with dynamic branching and concurrent execution, Junjo lets you define
the application logic as an explicit graph of nodes and edges.

The SDK remains decoupled from any model provider. Application code performs
model calls, retrieval, API requests, and other domain work inside nodes. A
`Workflow` owns repeatable execution of a graph definition against an isolated
store, returning an execution result with a detached final state.
