# Junjo AI Studio Frontend

React + TypeScript frontend for the Junjo AI Studio web UI.

## Running

Primary workflow is the full hot-reload stack from the repository root:

```bash
docker compose up --build
```

From another terminal:

```bash
docker compose logs -f frontend
```

Local URLs are selected by `JUNJO_BUILD_TARGET`:
- `JUNJO_BUILD_TARGET=development`: frontend `http://localhost:26151`, backend `http://localhost:26154`
- `JUNJO_BUILD_TARGET=production`: frontend `http://localhost:26153`, backend `http://localhost:26154`

For frontend-only testing/debugging outside Docker:

```bash
cd frontend
npm install
npm run dev
```

Vite serves on `http://localhost:26151` by default. This path is intended for frontend-focused testing/debugging and assumes the backend is reachable at `http://localhost:26154`. The supported full-stack workflow is `docker compose up --build` from the repository root.

## Commands

```bash
cd frontend
npm run test:run
npm run lint
npm run build
```

## Notes

- Browser API requests are explicit and cross-origin in development. The frontend talks to the backend URL directly.
- Production builds read `window.runtimeConfig.API_HOST` from `config.js`, generated by `frontend/prod-startup.sh`.
